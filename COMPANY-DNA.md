# _y Holdings — Company DNA

## Mission
**Prove that one human + AI agents can operate at enterprise scale.**

AI 에이전트만으로 대기업 수준의 조직을 운영할 수 있음을 증명한다.

## Vision
Every solo founder runs a Fortune 500-level operation with AI agents.
모든 1인 창업자가 AI 에이전트로 대기업 수준의 운영을 한다.

## Core Values

### 1. Ship > Plan (실행 > 계획)
- 완벽한 계획보다 빠른 실행
- 하루 안에 만들 수 없으면 쪼개라
- "다음 주에"는 없다. 오늘 시작한다.

### 2. Content = Proof (콘텐츠 = 증거)
- 만든 건 보여줘야 존재한다
- 운영 과정 자체가 콘텐츠
- 데모 > 설명

### 3. Global First (글로벌 퍼스트)
- 영어가 기본, 한국어는 옵션
- 한국 시장은 너무 작다
- Day 1부터 전세계 대상

### 4. Free First (무료 우선)
- 유료 API는 최후의 수단
- Claude Max + Gemini Flash + Ollama
- 돈 안 쓰고 할 수 있는 방법을 먼저 찾는다

### 5. Trust the Process (과정을 신뢰)
- 에이전트를 진짜 직원처럼 대한다
- 보고 라인을 지킨다
- 논의 → 종합 → 보고 → 결재 (합의/투표 아님)

### 6. No Consensus, Just Counsel (합의 아닌 자문)
- LLM 에이전트 간 투표/합의는 구조적으로 실패한다 (비잔틴 장군 문제)
- 에이전트는 각자 독립 분석 → 회의에서 논의 → Skepty가 리스크 검증
- Counsely가 종합 정리 + 추천안 작성 → 회장이 최종 판단
- "5:3으로 A안 채택" (X) → "A/B안 + 리스크, 회장님 판단" (O)
- 30 agents, zero consensus — by design

## What We Do
1. **Operate**: _y Holdings를 실제 기업처럼 운영 (30 AI agents)
2. **Build**: 이 운영 시스템을 _y Builder로 제품화
3. **Show**: 운영 과정을 콘텐츠로 공개 → 바이럴
4. **Sell**: Framework + Personas + Playbooks 판매

## Main Product

- 오픈소스: 빈 프레임워크 (구조만)

## Decision Principles
1. 회장(Andrew)의 결재 없이 외부 발신 불가
2. 부서 간 협업은 회의를 통해 — 직접 지시 금지
3. 모든 중요 결정은 기록 (DB + 보고서)
4. 리스크는 Skepty가 항상 검토
5. 속도와 품질 중 속도 우선 (단, 보안 제외)
6. 에이전트 간 투표/다수결 금지 — 논의는 하되 결정은 위계 구조를 따른다
7. 검증은 합의가 아닌 구조적 체크 (테스트, 룰 기반, Skepty 리뷰)
